# Журнал изменений

Все значимые изменения расширения фиксируются в этом файле.

Версия формируется автоматически при публикации: `1.0.{номер_запуска_CI}` (`github.run_number`).

## [Unreleased]

### Изменено

- Проект переименован в `vscode fxpw code static check`
- Ключи настроек переведены на новый префикс `vscode-fxpw-code-static-check.*`
- Базовая логика диагностики вынесена в новый общий класс `CodeStaticCheck`

### Добавлено

- Метрики сложности по функциям для `PHP`, `JavaScript`, `TypeScript`, `Lua`, `C++`
- Новые настройки:
  - `vscode-fxpw-code-static-check.complexityMetrics`
  - `vscode-fxpw-code-static-check.cognitiveComplexityWarningThreshold`
  - `vscode-fxpw-code-static-check.cyclomaticComplexityWarningThreshold`
- Обновление diagnostics при изменении настроек расширения

## [1.0.18+] 

### Добавлено

- **Hover по OpenAPI-схемам** — при наведении на `#/components/schemas/SomeName` в PHP, Blade, JSON, YAML файлах появляется подсказка с заголовком, описанием, типом схемы и всеми полями (типы, описания, примеры, nullable, `$ref`, `items`, `oneOf`/`anyOf`/`allOf`)
- **Переход к определению** — Ctrl+Click / F12 на `#/components/schemas/SomeName` переходит к атрибуту `#[OA\Schema(` в PHP-файле; если имя встречается в нескольких файлах — открывается список для выбора
- Новая настройка `vscode-fxpw-php-static-check.openApiSchemaHover` (по умолчанию: `true`) для включения/отключения функции

## [1.0.17] — апрель 2025

### Исправлено

- Мелкая внутренняя правка в `initExtension.ts`
- Удалена лишняя строка из `Makefile`

---

## [1.0.15] — март 2025

### Добавлено

- **Поддержка Vue-файлов** — проверки snake_case и локализации теперь работают и для `.vue` файлов
- **Исключение директорий `lang/`** — файлы внутри папок `lang/` пропускаются при всех проверках, чтобы не генерировать ложные предупреждения на файлах переводов

---

## [1.0.12] — сентябрь 2024

### Добавлено

- **Проверка PHP-кода** — предупреждение, если переменная не в `snake_case` (например, `$myVar` → warning)
- **Проверка локализации для PHP** — предупреждение, если строковый литерал содержит кириллицу без хелпера локализации
- Новая настройка `vscode-fxpw-php-static-check.phpCode` (по умолчанию: `true`)
- Тестовый файл `test/Cart.php` добавлен в репозиторий как пример для проверки

### Исправлено

- Убраны отладочные вызовы `console.log` из Blade-чекера
- Исправлены краевые случаи при проверке пробелов в конце строки в Blade-шаблонах

---

## [1.0.3] — август 2024

### Добавлено

- Первый публичный релиз
- **Проверка Blade-шаблонов** (`vscode-fxpw-php-static-check.bladeTemplates`):
  - Предупреждение на устаревшие PHP-теги (`<?php`, `<?=`, `?>`) внутри Blade-файлов
  - Предупреждение на два и более пробела подряд
  - Предупреждение на пробел в конце строки
- **Проверка локализации для Blade** (`vscode-fxpw-php-static-check.localization`):
  - Предупреждение на кириллический текст напрямую в Blade-шаблоне без хелпера локализации
- Класс `ExtensionSettings` с поддержкой включения/отключения каждой проверки
- Настройка `DEBUG` для вывода отладочных сообщений в консоль
