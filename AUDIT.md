# Ревизия проекта

## Найденные риски

1. `ComplexityAnalyzer` пока использует эвристический regex-based разбор, а не полноценные парсеры языков.
Это уже полезно для типовых функций, но на сложных сигнатурах, макросах, шаблонах, вложенных анонимных функциях и нетипичных форматированиях возможны ложные срабатывания или пропуски.

2. Базовые style-проверки по-прежнему специализированы в основном под `PHP`, `Blade`, `Vue`.
После переименования проекта это уже не выглядит как ошибка, но создаёт разрыв между общим брендом расширения и фактическим покрытием rule-based проверок.

3. Метрики сложности для `JavaScript`, `TypeScript`, `Lua`, `C++` добавлены без автоматических unit/integration tests.
Сейчас есть фикстуры для ручной проверки, но нет исполняемого тестового контура, который бы защищал от регрессий.

4. `OpenApiSchemaHoverProvider` остаётся изолированной подсистемой и пока не встроен в общую архитектуру анализаторов.
Это не ломает функциональность, но кодовая база всё ещё состоит из нескольких параллельных подходов к анализу.

## Что уже исправлено в этой итерации

- Проект переименован в `vscode fxpw code static check`
- Введены новые ключи настроек с префиксом `vscode-fxpw-code-static-check.*`
- Добавлен общий сервис `CodeStaticCheck`
- Добавлен мульти-язычный `ComplexityAnalyzer` для `PHP`, `JavaScript`, `TypeScript`, `Lua`, `C++`
- Добавлены warning-пороги для `Cognitive Complexity` и `Cyclomatic Complexity`
- Исправлен `lint`-скрипт на реальную директорию `ts/`
- Diagnostics теперь обновляются и при смене настроек
- Добавлены тестовые фикстуры для `JS`, `TS`, `Lua`, `C++`

## Следующие шаги

1. Вынести расчёт метрик в чистое ядро без зависимости от `vscode`, чтобы его можно было тестировать обычным test runner.
2. Добавить автоматические тесты на извлечение функций и вычисление метрик для всех поддерживаемых языков.
3. Разделить rule-based проверки по языкам на отдельные модули, как уже сделано для complexity-анализа.
4. Решить, нужен ли parser-based подход для `JS/TS` и `C++` на следующем этапе ради точности.
